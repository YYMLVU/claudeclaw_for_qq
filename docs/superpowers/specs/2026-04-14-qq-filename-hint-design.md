# QQ 文件名前置提示设计

## 目标
在 QQ Bot 发送文件给用户时，先发送一条简短文本，明确告诉用户即将收到的文件名，缓解 QQ 官方 Bot 富媒体消息不展示文件名/后缀的问题。

## 背景
当前 `msg_type: 7` 富媒体发送链路无法为接收端显式设置展示文件名。实际效果是用户收到文件时可能看不到文件名或后缀。由于这是平台能力限制，需要在机器人侧补一层可读提示。

## 方案
采用“发送前文件名提示”方案：
- 在每次实际发送文件前，先发送一条普通文本消息
- 文案格式固定为：`文件名：<basename>`
- 之后再调用现有 `uploadAndSendFile(...)`

## 范围
仅修改 QQ 发送结果文件的提示行为，覆盖：
- C2C
- Group
- Guild / channel

不修改：
- 文件选择逻辑
- 路径校验逻辑
- QQ 富媒体上传协议
- 失败提示文案

## 细节
1. 仅在文件已经通过路径校验、存在性校验、普通文件校验后发送文件名提示。
2. 如果文件发送最终失败，保留原有 `(文件发送失败: xxx)` 提示。
3. 多文件场景下，每个文件分别发送对应的文件名提示。
4. 提取一个小 helper 统一生成提示文本，避免三处手写重复文案。

## 测试
按 TDD 增加一个最小测试：
- helper 输入 `/home/xiao/tmp/report.pdf`
- 输出 `文件名：report.pdf`

然后将三处发送逻辑接入该 helper，并跑已有聚焦测试和构建验证。

## 风险
- 会多出一条文本消息，但这是当前平台限制下最直接、最稳定的补偿方案。
- 不尝试猜测客户端是否已经显示文件名，避免不可靠分支判断。
